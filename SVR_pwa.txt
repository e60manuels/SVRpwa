Je bent een coding assistent die een **PWA maakt van een bestaande Android WebView app**. Volg deze instructies strikt:

1. **Codebase**
- Gebruik de huidige Android WebView app codebase (Kotlin/JS) als input. De codebase staat in de map: C:\Users\emanu\AndroidStudioProjects\SVRcampings_v31
- Behoud exact dezelfde **UI/UX**: geen wijzigingen in knoppen, kleuren, iconen, layout of flow.
- Zet alle client-side logica en API calls (svr.nl API voor zoeken, filteren, detailpagina) om naar web-structuur (HTML, CSS, JS) geschikt voor PWA.
- Zorg dat toggles, kaarten, lijsten en detailpagina’s dynamisch laden zoals in de WebView app.

2. **Platform compatibiliteit**
- Installable op:
  - Android (Chrome / TWA)
  - iOS (Safari “Add to Home Screen”, beperkingen push-notificaties)
  - Windows (Chrome/Edge install)
- Zorg dat manifest en Service Worker geschikt zijn voor alle drie platforms, rekening houdend met beperkingen.

3. **Manifest & Service Worker**
- `manifest.json` volledig correct: naam, korte naam, start_url, display: standalone, background_color, theme_color, icons (512x512 + 192x192)
- Minimalistische Service Worker:
  - cache **alle statische assets**
  - **geen API caching**
  - offline fallback pagina (“Offline, probeer later opnieuw”)

4. **Deployment**
- PWA deployable via GitHub Pages
- Alle URLs in manifest en code correct voor GitHub Pages subfolder / HTTPS

5. **Scope & beperking**
- Geen nieuwe knoppen, kleuren, features of UX toevoegen
- Geen push-notificaties of background sync
- Focus puur op leerdoel: installable, manifest, service worker, multi-platform

6. **Output**
- Volledige PWA projectstructuur (HTML, CSS, JS, manifest.json, Service Worker), klaar voor GitHub Pages
- Exact dezelfde UI/UX als originele WebView app
- Geschikt voor Android, iOS en Windows install

PWA checklist:
1. Project setup
 Gebruik bestaande Kotlin/JS WebView codebase als input
 Maak een nieuwe webmap/projectstructuur: /index.html, /assets/, /js/, /css/
 Behoud exact dezelfde UI/UX (knoppen, kleuren, layout, toggle flows)

2. Manifest
 manifest.json aanwezig in projectroot
 Vul correct in:
"name": volledige app-naam
"short_name": korte naam
"start_url": relative path (bijv. /index.html)
"display": "standalone"
"background_color": zoals in app
"theme_color": zoals in app
"icons": 192x192 en 512x512 PNG
 Verwijzingen correct voor GitHub Pages (subfolder of root)

3. Service Worker
 Bestand: sw.js in root of /js/
 Cache alle statische assets: HTML, CSS, JS, images
 Niet cachen: API responses
 Offline fallback: simpele pagina (“Offline, probeer later opnieuw”)
 Registratie in JS: navigator.serviceWorker.register('/sw.js')

4. Platform checks
Android
 Chrome detecteert PWA-installable
 Full screen, standalone
 Kaart, lijst, toggles en detailpagina’s werken exact zoals WebView
iOS
 Safari “Add to Home Screen” werkt
 Full screen, standalone
 Beperkingen op push-notificaties noteren
 Test toggles, kaart en lijst functionaliteit
Windows
 Chrome/Edge detecteert PWA-installable
 Startmenu / taskbar snelkoppeling werkt
 Functionaliteit: Kaart, lijst, toggles, detailpagina’s OK

5. Deployment via GitHub Pages
 Project pushen naar repository
 GitHub Pages inschakelen (root of /docs of subfolder)
 Controleer HTTPS URL werkt
 Test installable + offline fallback

6. Doelen bevestigen
 PWA installatie op Android, iOS en Windows getest
 Service Worker caching en offline fallback getest
 Manifest correct en icons zichtbaar
 UI/UX exact hetzelfde als originele WebView app
