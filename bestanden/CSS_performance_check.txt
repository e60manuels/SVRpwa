Je CSS-bestanden blokkeren je render volledig. Dit is NIET "works-as-designed" - het is oplosbaar.

🔴 Het Probleem
Render-blocking CSS betekent: de browser wacht met tekenen totdat ALLE CSS binnen is:
1. HTML parser start
2. Ziet <link rel="stylesheet" href="unpkg.com/swiper.css">
3. ⏸️ PAUZEERT rendering
4. Wacht 1590ms op Swiper CSS van unpkg.com
5. Wacht 800ms op Leaflet CSS van unpkg.com
6. Wacht 1000ms op FontAwesome van Cloudflare
7. ✅ Nu pas eerste pixels tekenen
---
Totaal gewacht: ~3.4 seconden voor NIETS TE ZIEN

✅ De Oplossing: Critical CSS + Deferred Loading
Strategie:

Critical CSS = inline in <head> (layout, above-the-fold)
Non-critical CSS = asynchroon laden (Swiper, Leaflet, icons)


🎯 Concrete Implementatie
1. Maak Critical CSS Inline
In je <head>:
html<style>
/* Critical CSS - alleen wat direct zichtbaar is */
body { 
    margin: 0; 
    font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
}

#map { 
    width: 100vw; 
    height: 100vh; 
    position: fixed; 
    top: 0; 
    left: 0; 
}

/* Splash screen basics */
#splash { 
    position: fixed; 
    inset: 0; 
    background: white; 
    z-index: 9999; 
}
</style>
2. Laad Non-Critical CSS Asynchroon
Vervang je huidige <link> tags:
html<!-- ❌ OUD (blokkerend): -->
<link rel="stylesheet" href="https://unpkg.com/swiper@12.1.0/swiper-bundle.min.css">
<link rel="stylesheet" href="https://unpkg.com/leaflet@1.9.4/dist/leaflet.css">
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css">

<!-- ✅ NIEUW (non-blocking): -->
<link rel="preload" href="https://unpkg.com/swiper@12.1.0/swiper-bundle.min.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
<noscript><link rel="stylesheet" href="https://unpkg.com/swiper@12.1.0/swiper-bundle.min.css"></noscript>

<link rel="preload" href="https://unpkg.com/leaflet@1.9.4/dist/leaflet.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
<noscript><link rel="stylesheet" href="https://unpkg.com/leaflet@1.9.4/dist/leaflet.css"></noscript>

<link rel="preload" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
<noscript><link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css"></noscript>
Of gebruik dit helper script:
html<script>
// Async CSS loader
function loadCSS(href) {
    const link = document.createElement('link');
    link.rel = 'stylesheet';
    link.href = href;
    document.head.appendChild(link);
}

// Laad na DOMContentLoaded
document.addEventListener('DOMContentLoaded', () => {
    loadCSS('https://unpkg.com/swiper@12.1.0/swiper-bundle.min.css');
    loadCSS('https://unpkg.com/leaflet@1.9.4/dist/leaflet.css');
    loadCSS('https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css');
});
</script>

⚠️ Let Op: FOUC (Flash of Unstyled Content)
Als je CSS asynchroon laadt, kan je even onstyled content zien. Oplossing:
html<style>
/* Toon splash totdat alles geladen is */
#splash {
    position: fixed;
    inset: 0;
    background: white;
    z-index: 9999;
    display: flex;
    align-items: center;
    justify-content: center;
}

/* Verberg zodra CSS geladen is */
body.css-loaded #splash {
    display: none;
}
</style>

<script>
let cssLoaded = 0;
const totalCSS = 3; // Swiper, Leaflet, FontAwesome

function checkAllCSSLoaded() {
    cssLoaded++;
    if (cssLoaded === totalCSS) {
        document.body.classList.add('css-loaded');
    }
}

function loadCSS(href) {
    const link = document.createElement('link');
    link.rel = 'stylesheet';
    link.href = href;
    link.onload = checkAllCSSLoaded;
    document.head.appendChild(link);
}

document.addEventListener('DOMContentLoaded', () => {
    loadCSS('https://unpkg.com/swiper@12.1.0/swiper-bundle.min.css');
    loadCSS('https://unpkg.com/leaflet@1.9.4/dist/leaflet.css');
    loadCSS('https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css');
});
</script>
